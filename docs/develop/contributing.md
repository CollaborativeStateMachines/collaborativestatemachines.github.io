# Contributing to CSM

We welcome contributions to the Collaborative State Machines project. Whether you are addressing technical
issues, proposing extensions to the programming model, or improving documentation, your input helps advance
the ecosystem.

---

## Areas of Focus

### **Cirrina Runtime System**

Cirrina is the open-source reference runtime for executing CSM applications across the computing continuum.
Developed in **Kotlin**, the project is hosted on
[GitHub](https://github.com/CollaborativeStateMachines/Cirrina) and focuses on high-performance execution and
edge-cloud interoperability.

### **Research and Formalism**

As a research-centric initiative, we encourage contributions that strengthen the theoretical foundations of
the CSM model. For context on current research directions, please refer to our
[Publications](../research/publications.md).

### **Documentation and Tutorials**

Clear technical communication is vital for the adoption of declarative coordination models. We appreciate
contributions that improve our guides, API references, and step-by-step tutorials.

---

## Submission Process

1. **Issue Tracking**: For feature proposals or bug reports, please open a
   [GitHub Issue](https://github.com/CollaborativeStateMachines/Cirrina/issues). This allows for initial
   alignment with the maintainers before implementation begins.
2. **Pull Requests**: Contributions should be submitted via pull requests from a branched fork. Detailed
   technical requirements are available in the
   [Contribution Guide](https://github.com/CollaborativeStateMachines/Cirrina/blob/develop/CONTRIBUTING.md).
3. **Community Discussions**: For architectural inquiries or roadmap suggestions, please use the
   [Discussion Board](https://github.com/orgs/CollaborativeStateMachines/discussions).
